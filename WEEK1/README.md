# COM109 Client-Side Development

## Week 1: Introduction to WWW, HTML and GitHub

Welcome to **COM109 Client-Side Development**. This module introduces the core technologies used to build modern websites and interactive web applications.

Across the module, students will work with:

- **HTML** for page structure and content
- **CSS** for layout, colour, typography and responsive design
- **JavaScript** for interactivity and browser behaviour
- **jQuery** as an introduction to using JavaScript libraries

Week 1 focuses mainly on **HTML and web foundations**. Before styling or programming a website, students need to understand what the Web is, how browsers and servers communicate, how HTML structures information, and how developers use GitHub to manage source code.

## Learning Outcomes

By the end of Week 1, students should be able to:

- Explain the difference between the Internet and the World Wide Web.
- Describe what happens when a user enters a URL into a browser.
- Identify the roles of browsers, servers, HTTP and HTTPS.
- Create a basic HTML page using common tags.
- Use semantic HTML elements such as `header`, `nav`, `main`, `section` and `footer`.
- Understand how HTML, CSS and JavaScript work together.
- Explain why GitHub and source control are important in software development.
- Build and share a first developer portfolio webpage.

## What Students Will Build

Students will build a simple **HTML-only developer portfolio website**. The page will include:

- A name and page title
- An about section
- A skills list
- A projects section
- Useful links
- A contact form
- A footer

This is intentionally kept HTML-only in Week 1 so students can focus on structure before moving into CSS and JavaScript.

## Teaching Note

Use this week to make the Web feel practical and visible. Students already use services like Netflix, Amazon, TikTok, BBC, Uber, Deliveroo, Monzo and Revolut. The aim is to connect those everyday experiences to the technologies developers use to build them.
